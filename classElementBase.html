Detailed Description

template<CellType C>
class ElementBase< C >

Common base class for all element types.

At first I wanted to create only one template class for all elements (frac,tri,quad). While the tri and quad elements could be perfectly programmed in one template class, the fracture element was a bit different. So first I created the ElementBase class, which is common between fractures and poly (tri, quad) elements and then inheritted ElementPoly class from it. The ElementPoly class is specialized for the fracture elements, but is common between quads and triangles.

Note
The retuned value is a reference value. It means that there is a static (much like global) arma::rowvec member of ElementBase class and calling this function will change that static member and return a reference value to it. So this is a right usage: 
arma::vec *vec1,*vec2;
vec1 = &elem.lDatCnDis(lw,1);
vec2 = &elem.lDatCnDis(ln,2);
While this is a wrong one and makes the value of vec1 invalid: 
vec1 = &elem.lDatCnDis(lw,1);
vec2 = &elem.lDatCnDis(ln,1);
Note that if you do not use pointers everything will be ok since the data will be copied. However this will take additional run-time
